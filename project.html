Project Details – Secure Static Website on AWS

Detailed breakdown of how this static website is hosted securely using Amazon S3 and Amazon CloudFront.

Architecture Overview

  • Static HTML/CSS files hosted in a private S3 bucket
  • CloudFront distribution configured with the S3 static website endpoint as origin
  • Origin Access Control (OAC) used so only CloudFront can access S3
  • Bucket policy denies direct public access and allows only CloudFront
  • HTTPS enabled on CloudFront for secure content delivery

Deployment Steps

  1. Create an S3 bucket and enable static website hosting.
  2. Upload HTML, CSS and assets (images, icons, etc.).
  3. Keep bucket private – block all public access.
  4. Create a CloudFront distribution pointing to the S3 website endpoint.
  5. Configure Origin Access Control (OAC) and attach to the origin.
  6. Update S3 bucket policy to allow only the CloudFront OAC.
  7. Enable HTTP → HTTPS redirection in CloudFront behaviors.
  8. Test the website using the CloudFront domain URL.

Security Highlights

  • No direct public access to the S3 bucket.
  • All requests go through CloudFront, which acts as a secure front layer.
  • HTTPS ensures data in transit is encrypted.
  • Bucket policy + OAC combination protects your content.

Use Cases

This architecture is ideal for:

  • Portfolio websites
  • Static documentation sites
  • Landing pages for products
  • Any static front-end hosted without servers
